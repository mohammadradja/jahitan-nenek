# AGENTS.md

## Purpose

The `docs` folder explains the actual implementation for future developers and AI agents.

## Responsibilities

- Keep architecture, routes, schema, models, controllers, frontend, deployment, testing, and known issues accurate.

## Rules

- Document observed behavior only.
- Link related docs when useful.
- Keep Mermaid diagrams valid.

## Best Practices

- Update docs in the same change as implementation changes.
- Record known issues plainly instead of hiding them.

## Allowed Changes

- Documentation updates, diagrams, and agent guidance.

## Forbidden Changes

- Do not document imagined features as implemented.
- Do not include secrets from local environment files.

## AI Agent Instructions

- Re-inspect source files before changing factual docs.
- Keep `docs/README.md` as the index.
